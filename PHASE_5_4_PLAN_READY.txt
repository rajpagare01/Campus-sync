🔄 PAYMENT FLOW DIAGRAMS
═══════════════════════════════════════════════════════════════════════════════

Creating Order & Initiating Payment:
  User → /api/payments/create
     ↓
  OrderService: Create Order (status: PENDING)
     ↓
  PaymentService: Create payment intent/order on gateway
     ↓
  API Response: {orderId, clientSecret/paymentLink}
     ↓
  User: Redirected to payment form

Confirming Payment (via Webhook):
  Payment Gateway → Webhook: POST /api/payments/webhook/{gateway}
     ↓
  Validate: Verify webhook signature
     ↓
  PaymentService: Extract payment details
     ↓
  OrderService: Update Order (status: COMPLETED)
     ↓
  RegistrationService: Create event registrations
     ↓
  EmailService: Send confirmation email
     ↓
  Success: Order complete, user can access event

Requesting Refund:
  User → /api/orders/{orderId}/refund
     ↓
  OrderService: Validate refund eligibility
     ↓
  PaymentService: Process refund on gateway
     ↓
  OrderService: Update Order (status: REFUNDED)
     ↓
  RegistrationService: Cancel event registrations
     ↓
  EmailService: Send refund confirmation
     ↓
  Success: Refund processed (2-5 business days)

🔐 SECURITY MEASURES
═══════════════════════════════════════════════════════════════════════════════
✅ PCI-DSS Compliance
   - Zero storage of card data
   - All card processing via payment gateway
   - Encryption in transit (HTTPS)

✅ Webhook Security
   - Signature verification on every webhook
   - Timestamp validation (prevent replay attacks)
   - Webhook logs for audit trail

✅ Payment Security
   - Idempotency keys to prevent duplicate charges
   - Timeout handling with proper state recovery
   - Network failure recovery

✅ API Security
   - Rate limiting on payment endpoints (10 req/min per user)
   - CSRF protection
   - Input validation on all fields
   - Authentication required for all endpoints

✅ Data Security
   - API keys in environment variables (never hardcoded)
   - Sensitive data encrypted in database
   - Audit logging for all operations

✅ Error Handling
   - No sensitive info in error messages
   - Proper HTTP status codes
   - Structured error responses
